# grok-pi 资源准入策略与 Pi RPC 启动诊断

> 让 grok-pi 在启动 Pi RPC 子进程前，通过宿主策略过滤不兼容的 Pi-TUI renderer 扩展（如 pi-tool-display），并在 bootstrap 失败时展示 stderr 诊断而非仅显示 "stdout closed"。

## 背景与目的 (Why)

Pi RPC 子进程启动期间的 stderr 仅进入 tracing，bootstrap 失败时用户只看到 `Pi RPC stdout closed`，无法定位原因。`pi-tool-display` 会重注册 7 个内置工具并 monkey-patch `pi.registerTool`，与 grok-pi 的 RPC 工具投影不兼容，导致启动崩溃。

Pi 0.80.10 支持 `--no-extensions/--no-skills/--no-prompt-templates/--no-themes` 加显式重复资源参数，禁用自动发现后显式资源仍会加载。这为宿主侧资源准入提供了官方接缝，无需修改 Pi 源码。

## 变更内容概述 (What)

1. **新增 `pi_resource_policy.rs`**：ResourcePolicy + PiLaunchPlan，默认黑名单（pi-tool-display 等）、renderer 名称启发式、用户 allow/block 覆盖。
2. **接入 `grok-pi.rs` 启动路径**：PiResourceCatalog 发现 → 策略评估 → `--no-*` + 显式路径传入 Pi 子进程。
3. **`pi_rpc.rs` stderr 环形缓冲区**：32 行容量，bootstrap 失败时显示退出码 + stderr 尾部。
4. **修复 pager crate 预存测试编译错误**：PickerState.query private、PickerRow 缺 label_color、Color 未导入。

## 风险与影响评估 (Risk Assessment)

- 策略默认阻止 pi-tool-display 等已知冲突扩展；用户可通过 `ResourcePolicy.allow` 显式豁免。
- `--no-*` 标志改变了 Pi 的资源发现行为：所有资源必须由宿主显式传入。若 catalog 发现遗漏某资源，该资源将不会加载。
- 未信任项目资源仍由 Pi trust 语义裁决，策略不能绕过 trust。

## 回滚方案 (Rollback Plan)

回退 `grok-pi.rs` 中的策略接入代码（删除 `PiResourceCatalog::load` 和 `launch_plan` 相关行），恢复 Pi 自动发现。stderr 环形缓冲区为纯增量，无需回退。

## 详细变更说明

### 1. 资源准入策略（pi_resource_policy.rs）

**问题：** Pi-TUI renderer 扩展（如 pi-tool-display）在 grok-pi RPC 环境下无价值且会导致启动崩溃。

**方案：** 三层过滤：
- 默认黑名单：`npm:pi-tool-display`、`npm:pi-custom-header`、`npm:pi-custom-footer`
- Renderer 名称启发式：文件名/父目录名包含 `tool-display`、`custom-header`、`custom-footer`
- 用户覆盖：`ResourcePolicy.allow` / `ResourcePolicy.block` 优先于默认规则

**影响范围：** 仅影响 grok-pi 启动路径；Pi 交互模式不受影响。

### 2. stderr 环形缓冲区（pi_rpc.rs）

**问题：** bootstrap 失败时仅显示 `Pi RPC stdout closed`，无法定位扩展加载错误。

**方案：** 32 行环形缓冲区收集 Pi 子进程 stderr；stdout EOF 或进程退出时，将 stderr 尾部附加到 pending 请求的失败消息中。

**影响范围：** 仅影响错误消息内容；正常运行无额外开销。

### 3. 启动路径接入（grok-pi.rs）

**问题：** Pi 自动发现会加载所有已安装扩展，包括不兼容的 renderer。

**方案：** 在 `pi_args_with_startup_flags()` 之后、bridge 扩展循环之前插入策略评估。使用 `PiResourceCatalog::load()` 发现资源，`ResourcePolicy::evaluate()` 过滤，`PiLaunchPlan` 生成 `--no-*` + 显式路径参数。Bridge 扩展（subagent、bash、recap 等）在策略之后单独追加，始终加载。

**影响范围：** grok-pi 启动参数变化；用户 `--no-extensions` / `--no-skills` CLI 标志仍被尊重。
