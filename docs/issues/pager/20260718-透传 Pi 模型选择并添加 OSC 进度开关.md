# Issue: 透传 Pi 模型选择并添加 OSC 进度开关

## Goal

让 external Pi profile 的裸 `/model` 直接打开 Grok 原生模型选择器，并将当前模型置顶；同时在 F2 设置中提供 OSC 9;4 terminal tab progress 开关，并默认关闭。

## 背景/问题

当前 `/model` 需要先进入输入框参数搜索流程。评估 Pi Remote TUI 后，用户选择复杂度兜底：保留 Grok 原生 picker，但裸命令直接打开且当前模型置顶。现有 OSC 9;4 progress 功能只由 `[ui.notifications].progress_bar` 配置，默认开启且未进入 F2。

## 验收标准 (Acceptance Criteria)

- [x] WHEN external Pi profile 执行裸 `/model`，系统 SHALL 直接显示 Grok 原生模型选择器。
- [x] WHEN `/model <provider/model>` 执行，系统 SHALL 保留可直接切换模型的既有语义。
- [x] WHERE F2 设置，系统 SHALL 显示 “Show OSC 9;4 progress indicators in the terminal tab bar” 布尔开关。
- [x] WHEN 无用户配置，OSC 9;4 progress SHALL 默认关闭。
- [x] WHEN 开关运行时变化，NotificationService SHALL 立即应用并在关闭时清理已激活 indicator。

## 实施阶段

### Phase 1: 规划和准备
- [x] 分析 `/model`、Remote TUI、F2 registry 与 NotificationService 调用链
- [x] 确认只通过既有 Remote TUI bridge，不创建第二个终端 UI

### Phase 2: 执行
- [x] 让裸 `/model` 打开原生 picker，并将当前模型置顶
- [x] 添加 F2 progress_bar 设置、持久化和 live setter
- [x] 默认关闭 progress_bar，并同步文档

### Phase 3: 验证
- [x] 聚焦单元测试
- [x] `cargo test -p pi-grok-adapter`
- [x] `cargo check -p xai-grok-pager-bin --bin grok-pi`
- [x] diff 与 source-identity seam 审查（verifier 仍有仓库既存 baseline/语义 blocker，单独记录）

## 关键决策

| 决策 | 理由 |
|------|------|
| `/model` 保留 Grok 原生 picker | Pi ModelSelector 依赖 interactive 私有 SettingsManager/lifecycle，用户选择复杂度兜底 |
| OSC 开关映射现有 `progress_bar` 配置 | 不新增重复配置或第二套 progress 实现 |
| progress 默认 false | 用户明确要求默认关闭 |
