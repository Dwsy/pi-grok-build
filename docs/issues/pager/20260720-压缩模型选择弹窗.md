# Issue: 压缩模型选择弹窗

## Goal

将 `/model` 的模型列表维持在约 70% 终端高度，避免 compact prompt 设置将其扩展为近乎全屏的弹窗。

## 背景/问题

`ModalSizing::with_compact()` 会将垂直边距归零。模型列表同时有底部详情面板，compact 模式下会占满终端，违背紧凑选择器的交互预期。

## 验收标准

- [x] `/model` 初始模型列表的高度约为终端高度的 70%。
- [x] compact prompt 设置不改变该模型列表的高度约束。
- [x] 其他参数选择器继续沿用既有 compact sizing 行为。
- [x] `cargo test -p xai-grok-pager-bin --bin grok-pi` 通过。
- [x] `cargo check -p xai-grok-pager-bin --bin grok-pi` 通过。

## 实施

模型列表路径以整数饱和运算按终端高度计算 15% 的上下边距，并跳过 `with_compact()`；模型 effort 子步骤及其他参数选择器维持原逻辑。

## 关键决策

| 决策 | 理由 |
|------|------|
| 仅对初始模型列表跳过 compact sizing | 该路径有详情面板；不扩大其他弹窗改动范围。 |
| 使用整数饱和运算 | 避免浮点尺寸计算，并安全处理小终端。 |
