# ⚠️ 已废弃 — 见 `20260721-DEBUG-BRIEF-noncompact-lag.md`

本文档记录的修复方向（suppress sticky、freeze animation、throttle draw）已被证明是**错误的**。

真正的根因是 `render.rs` 中 `prepare_layout` 宽度振荡（每帧触发 Case 1 全量 rebuild），与 expanded Edit、sticky header、spinner 均无因果关系。

## 保留的有效改动

- `has_expanded_edit_in_viewport()`：保留（event_loop 120ms 节流仍引用）
- event_loop 120ms draw 下限：保留（无害防御）

## 已回退的错误改动

- `set_suppress_sticky_headers(!is_idle())`：删除
- `tick()` 中 `heavy_expanded_edit` 门控：回退
- `tick_demand()` 中 `heavy_expanded_edit` 门控：回退
