# Issue: 修复 live turn 中展开 Edit/Write 导致整页卡顿

## Goal

运行中展开（或自动展开）Edit/Write 可折叠区域时 TUI 不再持续卡顿；resume/idle 路径行为不变。

## 已确认根因

| 场景 | 行为 |
|---|---|
| Live turn + 展开 Edit 在视口 | 卡（折叠后立刻流畅） |
| Resume 同会话 + 展开/折叠 | 不卡 |
| `collapsed_edit_blocks` | ON（默认折叠；卡顿来自手动/untrusted 展开） |

Live 路径在 `TurnRunning` 时：

1. `tick_demand` 因 `!is_idle()` 保持 `TickDemand::Fast`（~30fps）
2. `tick()` 因 status spinner / running wave 强制整屏 `needs_redraw`
3. 展开 Edit 每帧重绘大块 styled diff → 输入与滚动被饿死

Resume 为 Idle，无上述热循环，故同样内容不卡。

## 修复

- `ScrollbackState::has_expanded_edit_in_viewport()`：检测视口内带 hunk/error 的 Expanded Edit
- `AppView::tick`：有重型展开 Edit 时，不因 scrollback 动画 / turn spinner 请求重绘
- `AppView::tick_demand`：同上，不单为 turn status / wave 维持 30fps
- ACP 流式：视口有展开 Edit 时 `min_draw_interval` 下限 120ms

内容更新、用户输入、交互路径仍正常请求绘制。
