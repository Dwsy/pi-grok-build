# Issue: 整合指定上游提交

## Goal

将上游 Grok Build commit `8adf9013a0929e5c7f1d4e849492d2387837a28d` 的生产 Pager 变更整合到 Pi-Grok 原生 TUI 交付中，并重新移植 headless Pi adapter 接缝，保持 Pi 不修改且所有 UI 仍由 Grok Pager 渲染。

## 背景/问题

当前交付将 Grok Build 作为 `grok-build-main/` 的 vendor 子树；指定上游 commit 是独立上游仓库根，二者没有 Git 共同祖先，不能直接 merge。必须以干净上游 worktree 为基线重新应用集成 patch，并手工解决上游 event-loop 漂移，然后再将结果导入交付树。

## 验收标准 (Acceptance Criteria)

- [ ] WHEN 上游 commit 被导入，系统 SHALL 保留该 commit 的 Pager 行为并重新构建 `grok-pi`。
- [ ] WHERE Pi profile 启动，系统 SHALL 继续通过 `run_external` 使用上游生产 Pager，而不创建第二个 TUI。
- [ ] IF 上游与 adapter 接缝冲突，THEN 仅在声明的 ACP/外部 profile 接缝解决，并以构建、mock、架构审计验证。
