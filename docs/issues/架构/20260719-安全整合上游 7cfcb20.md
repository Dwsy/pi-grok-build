# Issue: 安全整合上游 7cfcb20

## Goal

在不破坏 `grok-pi` 的 Pi-Grok 架构边界与既有功能前提下，安全吸收 upstream `7cfcb20` 的 Pager 更新。

## 背景/问题

本地 `main` 以 upstream `98c3b24` 为基线，已经累积 41 个 Pi-Grok 集成提交；upstream `7cfcb20` 是一次大规模 monorepo 同步，覆盖 Pager ACP、事件循环、状态、modal 与 picker。根提交会把本地窄接缝卷入广泛冲突，故禁止直接在主工作树合并。

## 验收标准 (Acceptance Criteria)

- [ ] 上游变更在隔离 worktree 中完成评估与整合；主工作树在验证前不受影响。
- [ ] Pager 保持唯一可见 TUI，Pi 保持唯一 agent core，`pi-grok-adapter` 仍为 headless library-only。
- [ ] 保留并验证 Pi 的会话、队列、context、slash、通知、子代理与后台 Bash 接缝。
- [ ] 合并结果通过最窄相关测试、`cargo check -p xai-grok-pager-bin --bin grok-pi`、`git diff --check` 与范围审查。

## 关键决策

| 决策 | 理由 |
|------|------|
| 不直接 merge 到 `main` | upstream 更新横跨 Pager 核心，直接冲突解决会破坏可回退性。 |
| 用隔离 worktree 试合并 | 主工作树保持可运行的 `7de7259` 基线。 |
| 窄接缝 reapply，而非全量上游替换 | 维持 Pager-only TUI、Pi-only core、adapter headless 等架构不变量。 |
