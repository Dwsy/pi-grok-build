# 项目级 `.grok-pi` 隔离

## 决策

grok-pi 在用户态与项目态均与 stock Grok 隔离：

| 层级 | stock Grok | grok-pi 默认 |
|---|---|---|
| 用户 home | `~/.grok` | `~/.grok-pi`（`$GROK_HOME`） |
| 项目树 | `<repo>/.grok` | `<repo>/.grok-pi`（`$GROK_PROJECT_DIR`） |

## 机制

- `xai_grok_config::project_config_dirname()` / `project_config_dir(root)`
- 解析：`$GROK_PROJECT_DIR`（可省略前导 `.`）→ 默认 `.grok`
- `grok-pi` 启动 `ensure_default_grok_home()`：未设置时注入 `GROK_PROJECT_DIR=.grok-pi`
- 生产路径（workflows / hooks / config / skills / plugins / agents / sandbox / LSP / checkpoints / personas…）走 helper

## 兼容

- 单测与 stock 路径默认仍为 `.grok`（不设 env）
- **不**自动读取 `<repo>/.grok`（真正隔离）；需迁移时手动拷到 `.grok-pi`
- 覆盖：`export GROK_PROJECT_DIR=.grok` 可临时回到 stock 项目目录名

## 验证

- `cargo check -p xai-grok-pager-bin --bin grok-pi`
- `cargo test -p xai-grok-shell --lib session::workflow`（1 预存 macOS `/private` 符号链接失败，与本改无关）

## 与 Workflow 的关系

- 用户脚本：`~/.grok-pi/workflows/*.rhai`
- 项目脚本：`<repo>/.grok-pi/workflows/*.rhai`
- slash：`/workflow` `/workflows` `/create-workflow`（见 [上游 Workflow Issue](./20260722-上游Workflow完整兼容-Pi-Spawn后端.md)）
- F2：`[ui].pi_workflows` 写在 `~/.grok-pi/config.toml`
