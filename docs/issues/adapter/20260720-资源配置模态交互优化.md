# Issue: 资源配置模态交互优化

## Goal

使 `/pi-config` 在大量 Pi 资源下沿用 Grok Pager 原生 MCP/插件设置的可预测交互：状态过滤、增量搜索、Global/Project tab 正反切换和明确的结果计数。

## Baseline

- `views/extensions_modal.rs` 的 `StatusFilter`：`All → Enabled → Disabled → All`，`f` 循环。
- 原生 modal chrome 已处理 tab 点击；`PiConfigModalState::select_tab()` 是其切换接缝。
- 搜索结果应展开匹配资源源，但状态过滤不改变用户手动折叠状态。

## Scope

- 修改 `crates/codegen/xai-grok-pager/src/views/pi_config.rs`，以及恢复 Settings modal 的 `Enter → OpenPiConfig` 导航接缝。
- 添加 `All` / `Enabled` / `Disabled` 资源过滤，`f` 循环切换。
- 搜索匹配过滤后的资源身份、类型、来源、路径及状态语义。
- `Tab` / `Shift+Tab` 在 trusted project 的两个 scope 间正反切换；tab 点击沿用原生 modal chrome。
- 显示活动 filter 与“匹配数 / scope 总数”，更新快捷键和单元测试。

## Acceptance Criteria

- [x] `f` 按 All、Enabled、Disabled 循环，树和结果计数同步更新。
- [x] `/` 搜索只显示当前 filter 下的资源，且搜索时展开匹配源。
- [x] `Tab` 与 `Shift+Tab` 均能切换 Global/Project，搜索/filter 状态可预测。
- [x] 资源状态不依赖颜色单独表达；文字 filter label 和计数可见。
- [x] 定向测试、grok-pi binary check 和 diff 检查通过。
