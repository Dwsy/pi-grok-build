# Issue: 资源目录发现与启用语义对齐

## Goal

让 grok-pi 的 Rust 资源目录与 Pi 0.80.10 `DefaultPackageManager.resolve()` 对同一份 `settings.json` 得到一致的全局资源集合、来源和 enabled 状态；这套目录同时是 `ResourcePolicy` 生成显式 Pi CLI 资源参数的事实来源。

## 已确认事实

- Pi 把 `git:github.com/owner/repo` 解析为 Git 包并从 `~/.pi/agent/git/github.com/owner/repo` 加载。
- 当前 Rust `package_dir()` 不识别 `git:` 前缀，误将该输入当作本地路径；因此绝大多数 GitHub 包未进入目录。
- 对 package object，Pi 区分：
  - 没有某类过滤字段：按 manifest（或约定目录）默认启用；
  - 某类字段为 `[]`：显式展示该类型全部资源，但全部禁用；
  - 有 `+`/`-` 过滤：先枚举 Pi manifest/目录所允许的候选，再按过滤结果决定 enabled。
- Pi 包一旦存在 `package.json.pi`，只加载 manifest 明示的资源类别；不会把未声明类别退化为 `index.ts` 或约定目录。

## Scope

- 修改 `crates/codegen/xai-grok-pager/src/pi_resource_config.rs`。
- 对齐 user/global 包的 git source 解析、manifest 资源枚举和 package filter enabled 状态。
- 增加不依赖用户真实目录的回归测试，覆盖 `git:`、HTTPS/SSH、空 filters、manifest 未声明类型和过滤状态。
- 不修改 Pi 核心、RPC 或 Pager 渲染架构。

- [x] 包中的 resources 数量与 Pi manifest/默认约定规则一致，不把未声明类型误扫入。
- [x] package filter 的缺省、空数组、`+` 和 `-` 语义与 Pi 一致；目录仍保留 disabled 资源供 TUI 切换。
