# Issue: 提取 Pi 上下文投影模块

## Goal

将 `pi_adapter.rs` 中 Pi context stats、消息 token 估算与原生 Grok `SessionInfoResponse` JSON 投影提取为私有纯模块，同时保持 context bar 和 `/context` 的已有语义。

## 背景/问题

在已拆出工具投影后，`pi_adapter.rs` 仍有 2,762 行。`1865–2186` 行是一个完整且纯粹的 context 投影群：解析 Pi `contextUsage`、估算 message/tool payload、缩放到权威 used tokens、构建 `x.ai/session/info` 响应，以及 assistant usage token 回退。

已有功能 Issue 已定义协议契约：Pi 是 `used/contextWindow` 的权威来源；`get_messages`/`get_entries` 只用于 breakdown 估算；Grok Pager 继续渲染原生 ContextInfoBlock。本轮只迁移纯映射，不移动 RPC 获取、缓存、ACP 更新或 ext response 封装。

不在本轮范围：`refresh_context_usage`、`note_context_tokens`、`handle_session_info` 的 RPC/状态编排，`PiAgent` 生命周期、session/tree、queue、QuestionView 映射以及 `model.rs`。

## 验收标准 (Acceptance Criteria)

- [x] `context_projection.rs` 只依赖 `model::{number, string, PiModel}` 与 serde JSON；不依赖 Pi RPC、ACP channel、terminal 或 Pager UI。
- [x] `pi_adapter.rs` 保留 Pi RPC 请求、`last_context_tokens` 缓存、session ID/CWD 采集、`ext_response` 和 ACP update；只调用私有 context 投影函数。
- [x] `contextUsage.tokens` 优先、null fallback、model context-window fallback、message/tool token 比例缩放、compaction count 与 native JSON 字段形状不变。
- [x] context 投影的五项单测随实现迁移；`cargo test -p pi-grok-adapter`、adapter/binary `cargo check` 和完整 `./build.sh` 成功。
- [x] 不修改 Pager、包内 Pi 或 adapter 公共 API；与已存在的 grok-pi/tool-projection 未提交改动保持隔离。

## 关键决策

| 决策 | 理由 |
|------|------|
| 只移动纯 context 映射 | RPC、缓存和 ACP 生命周期继续集中在 `PiAgent`，避免改变异步及状态语义。 |
| 模块保持 crate 私有 | 不扩大 adapter 对外 API；Grok/Pi 外部边界保持不变。 |
| `Pi` 的 stats 优先 | 这是既有 context 功能的权威数据来源，消息文本只能用于 breakdown。 |
| 测试与模块同置 | 复制 `tool_projection`、queue/todo bridge 的模块化模式，直接锁定协议 payload。 |
