# Issue: 实现 x.ai/session/info 以支持上下文点击渲染

## Goal

点击 Grok 原生 context bar / 运行 `/context` 时，应渲染原生 `ContextInfoBlock`，而不是报错：

```text
Couldn't load context info: couldn't fetch session info: Method not found: x.ai/session/info
```

## 根因

1. Pager 通过 ACP ext method `x.ai/session/info` 拉取 `SessionInfoResponse`。
2. Grok shell 后端实现了该方法；`pi-grok-adapter` 的 `ext_method` 未实现。
3. Status bar 的 `used` 已由 `_meta.totalTokens` 同步（见 status-bar issue），但完整 breakdown 点击路径仍依赖 session/info。

## 设计

| 边界 | 归属 |
|---|---|
| 可见 UI | Grok 原生 `ContextInfoBlock` / scrollback |
| 数据源 | Pi `get_session_stats`（权威 used/window）+ 可选 `get_messages`（字符估算 breakdown） |
| 映射层 | `pi-grok-adapter` headless only |

参考 pi-context 扩展（`~/.pi/agent/extensions/pi-context/src/context.ts`）：

- 用 `ctx.getContextUsage()` 拿 `tokens / contextWindow / percent`
- 用消息文本 `ceil(len/4)` 估 raw 分项
- 用 `ratio = actual / raw` 缩放，使分项之和等于权威 used

在 adapter 中：

- **不**再造 pi-tui overlay
- 把缩放后的分项写入 Grok `ContextInfo` 字段，让原生 block 渲染

### 字段映射

| Grok `ContextInfo` | Pi 来源 |
|---|---|
| `used` | `contextUsage.tokens`，fallback `last_context_tokens` |
| `total` | `contextUsage.contextWindow`，fallback model `context_window` |
| `usagePct` / `freeTokens` | 由 used/total 计算 |
| `messageTokens` | `get_messages` 文本估算后缩放 |
| `systemPromptTokens` / `toolDefinitions*` | Pi RPC 未暴露 → 0（差额进 Reasoning/overhead） |
| `turnCount` / `toolCallCount` / `messageCount` | `get_session_stats` 计数 |
| `compactionCount` | messages 中 compaction summary 计数 |
| `model` / `modelDisplayName` | bootstrap model |
