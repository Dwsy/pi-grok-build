# Issue: 补齐 Pi 可见会话事件映射

## Goal

让需要回看的 Pi 状态信息进入 Grok Pager 原生 scrollback：扩展信息通知可见且可回看，压缩开始、完成和失败复用原生 `SessionEvent` 块显示；不改变 Pi core，也不在 adapter 中创建 UI。

## 背景/问题

`/grok-cli-vision:status` 通过 Pi `ctx.ui.notify(..., "info")` 发送多行状态。当前 adapter 将其投影为 `pi/ui/notify`，Pager 只进入三秒 toast 队列，内容不会写入 scrollback。原生 Grok HookAnnotation 则会追加 `SessionEventBlock`，可稳定回看。

Pi compaction 事件当前仅映射为 `setStatus("Compacting context…")`；结束时清除 status，失败才 toast，缺少原生的压缩完成/失败消息块。

## 验收标准 (Acceptance Criteria)

- [ ] Pi 的信息类 extension 通知在保留原生 toast 的同时，追加为清晰标识来源的原生 scrollback 会话事件；warning/error 仍保持瞬态通知语义。
- [ ] Pi `compaction_start` / `auto_compaction_start` 与结束事件映射到 Pager 原生压缩 SessionEvent 块，完成块包含 Pi 实际可用的 token/耗时数据；字段缺失时不伪造数值。
- [ ] 错误压缩显示原生失败块，不与成功完成块重复。
- [ ] adapter 维持 headless；Pager 使用现有 scrollback/SessionEvent，而非新增 TUI。
- [ ] 补充覆盖通知与压缩映射的定向测试，并运行可用的 adapter/Pager 验证；既有 Pager test 编译 blocker 单列记录。

## 关键决策

| 决策 | 理由 |
|------|------|
| 信息级 notify 同时保留 toast 与新增回看事件 | `grok-cli-vision:status` 是用户显式查询结果，需要稳定可见；warning/error 保持现有紧急提示语义。 |
| 扩展事件经 adapter 的窄 ACP notification 投影 | Pi RPC 没有直接的 SessionEvent 结构；不修改 Pi core，adapter 仍只传递语义。 |
| Pager 复用 `SessionEventBlock` | 与原生 Hook/compaction 呈现一致，不新建第二套 TUI。 |
| 不根据通知文案或扩展名做字符串特判 | 避免对外部扩展版本和英文文本建立脆弱耦合。 |
