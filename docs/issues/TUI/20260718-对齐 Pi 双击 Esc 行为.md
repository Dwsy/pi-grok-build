# Issue: 对齐 Pi 双击 Esc 会话树入口

## Goal

让 `grok-pi` 的 External profile 复用 Grok Pager 原生 `SessionTree`，实现与 Pi interactive TUI 默认行为等价的空编辑器双击 `Esc` 会话树入口；不引入第二套 TUI 或修改 Pi RPC。

## 背景/问题

Pi interactive TUI 的默认 `doubleEscapeAction` 是 `tree`：普通模式、空编辑器、非流式/非 Bash 情况下，两次 `app.interrupt` 在严格小于 500ms 时打开会话树。当前 `grok-pi` 复用 Pager 默认行为：空 prompt 的双击 `Esc` 打开 Rewind picker，且没有将 Pi 默认行为映射到已存在的 `SessionTree`。

Pi 的 `/settings` 是独立入口，不是双击 `Esc` 的目标；不能用 Pager Settings modal 冒充 Pi Settings selector。Pi 的 `fork` / `none` 选项和可配置 `app.interrupt` 键位没有通过 RPC 投影给 Pager，因此本次只对 `grok-pi` 实现 Pi 默认的 `Esc → tree` 行为。

## 关键决策

| 决策 | 理由 |
|------|------|
| 以 Pi interactive 源码为行为事实源 | 真实逻辑是 `app.interrupt` 的双击，而非按名称猜测的 Settings 快捷键。 |
| 复用 Pager 原生 `SessionTree` | `grok-pi` 已实现 `/tree`，保持 Grok Pager 为唯一终端 UI。 |
| 仅对 External profile 生效 | 不改变 Grok product profile 的 Rewind 快捷键。 |
| 不将双击 Esc 映射到 Settings | Pi 的双击 Esc 默认打开会话树；`/settings` 是独立命令。 |
| 不实现 `fork` / `none` / 可配置键位 | Pi RPC 不暴露该设置、键盘事件或原始编辑器状态；本次只实现 Pi 默认语义。 |
| 不改 Pi RPC 或 Pi 源码 | 键盘事件属于 Pager 输入层，Pi 继续作为 agent core。 |
